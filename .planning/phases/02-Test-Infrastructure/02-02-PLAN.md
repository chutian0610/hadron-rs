  - `add_worker(info: WorkerInfo)` - add a worker
  - `register(host, port, slots)` - returns predictable UUID based on counter
  - `get_worker(id)` / `list_workers()` - accessors that match WorkerRegistry interface
  - `update_partition(worker_id, partition_id, table_name, file_path)` - helper to set up locality info
- The mock should return cloned WorkerInfo so tests can inspect and verify

**3. octopus-common/src/test_utils/fixture.rs**
Create TestRecordBatchFactory with builder pattern per D-04, D-05:
- Import arrow types: RecordBatch, ArrayRef, Schema, Field, DataType
- Implement `TestRecordBatchFactory` with builder pattern:
  - `new()` - start empty
  - `with_schema(schema: Schema)` - set schema directly
  - `add_column<T: IntoArrow>(name: &str, values: Vec<T>)` - add column from Vec
  - `add_integer_column(name: &str, values: Vec<i64>)` - helper for Int64Array
  - `add_string_column(name: &str, values: Vec<&str>)` - helper for StringArray
  - `build()` -> Result<RecordBatch, ArrowError>
- Provide factory methods for common schemas:
  - `orders_table()` -> TestRecordBatchFactory with order_id, amount, date columns
  - `users_table()` -> TestRecordBatchFactory with id, name, value columns
